Taxonomy assignment with HITdb

Christian Diener

When analyzing intestinal samples you might want to prefer to use a dedicated database which includes a curated set of microbes that are known to be present in the intestines. This database is provided by HITdb. The potential advantages are:

  • faster taxonomy assignment since the database is smaller
  • better taxonomy assignement, specifically on the species level (according to the publication)
  • easy detection of contamination since those sequence variants are classified as NA

The obvious disadvantage is that HITdb only works for intestinal samples.

Getting HITdb for dada2

If you use the mbtools HITdb is included as an extra file. You can get the file path with

hitdb <- system.file("extdata", "hitdb_v1.00.fa.gz", package = "mbtools")

If you are not using the package it can be obtained from Zenodo with the DOI 10.5281/zenodo.159205.

How to use it

We will load the sequence table we created in the mock example.

seqtab <- readRDS("svs.rds")

Now we assign the taxonomy using HITdb

library(dada2)
## Loading required package: Rcpp
taxa <- assignTaxonomy(seqtab, hitdb)
colnames(taxa) <- c("Phylum", "Class", "Order", "Family", "Genus", "Species")

And we are done :)
